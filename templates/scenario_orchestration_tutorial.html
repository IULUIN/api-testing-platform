🎭 场景编排教程

学习如何创建多步骤测试场景，实现复杂业务流程测试

📖 功能介绍

场景编排功能允许你将多个测试用例组合成一个完整的测试场景，按顺序执行。 这对于测试复杂的业务流程非常有用，比如：注册 → 登录 → 下单 → 支付。

💡 核心特性

  • 多步骤编排：将多个测试用例按顺序组合
  • 变量传递：从前一步提取数据，传递给后续步骤
  • 等待控制：每个步骤可设置等待时间
  • 失败处理：某步骤失败时，可选择继续或停止

🎯 适用场景

用户注册流程

注册 → 发送验证码 → 验证邮箱 → 完善信息

电商购物流程

登录 → 浏览商品 → 加入购物车 → 下单 → 支付

数据依赖测试

创建资源 → 查询资源 → 更新资源 → 删除资源

权限验证流程

获取Token → 访问受保护资源 → 刷新Token → 退出登录

🚀 创建测试场景

步骤1：进入场景管理

点击导航栏的"场景编排"菜单，进入场景管理页面。

步骤2：新建场景

点击"新建场景"按钮，填写场景名称和描述。

步骤3：添加测试步骤

从测试用例列表中选择要添加的用例，点击"添加步骤"。你可以：

  • 调整步骤顺序（拖拽或使用上下箭头）
  • 设置每个步骤的等待时间
  • 配置变量提取规则
  • 删除不需要的步骤

步骤4：配置变量提取

如果需要在步骤间传递数据，可以配置变量提取：

示例：从登录接口提取token

  • 变量名：auth_token
  • 提取路径：data.token
  • 使用方式：在后续步骤的请求头中使用 ${auth_token}

步骤5：保存并执行

配置完成后，点击"保存"按钮。然后可以点击"执行场景"开始测试。

🔗 变量提取与传递

什么是变量提取？

变量提取允许你从某个步骤的响应中提取数据，并在后续步骤中使用。 这对于处理依赖关系非常重要。

提取规则配置

字段 说明 示例
变量名 定义变量的名称 user_id, token, order_id
提取路径 JSON路径表达式 data.id, result.token
默认值 提取失败时的默认值 可选

变量使用方式

在后续步骤中，使用 ${变量名} 的格式引用变量：

在URL中使用：

https://api.example.com/users/${user_id}

在请求头中使用：

Authorization: Bearer ${auth_token}

在请求体中使用：

{"order_id": "${order_id}", "status": "paid"}

📊 场景执行与结果

执行场景

在场景列表页面，点击"执行"按钮即可运行场景。系统会按顺序执行所有步骤。

查看执行结果

执行完成后，你可以查看：

  • 整体状态：成功/失败/部分成功
  • 步骤详情：每个步骤的执行结果
  • 响应数据：每个步骤的请求和响应
  • 变量值：提取的变量及其值
  • 执行时间：每个步骤的耗时

执行历史

点击场景名称可以查看该场景的所有执行历史，方便追踪和对比。

💡 最佳实践

✅ 建议做法

  • 场景名称要清晰描述业务流程
  • 合理设置步骤间的等待时间
  • 为关键数据配置变量提取
  • 添加必要的断言验证
  • 定期检查场景执行结果

❌ 避免做法

  • 场景步骤过多（建议不超过10步）
  • 步骤间没有必要的依赖关系
  • 忽略错误处理和异常情况
  • 变量命名不规范或重复
